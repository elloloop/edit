Why edit exists

AI agents write code. You still need to read it.

When Claude, Copilot, or Cursor generate changes across your codebase, you need a fast way to navigate, review, and understand what happened. That’s the gap edit fills.

AI agents generate hundreds of lines of code in seconds. But the review step still happens in clunky browser-based diff viewers or bloated IDEs. You need something fast, terminal-native, and purpose-built for reading code.
The review bottleneck — the real cost of AI-assisted development
The best diff review experience shows you exactly what changed, in context, with syntax highlighting and line numbers. Side-by-side with the original. Not a wall of red and green in a terminal.
Git diff review — done right with a real editor, not a pager
Navigation should be instant. Type a function name and jump there. Type a line number and land on it. Search and cycle through results. No mouse required. No keybinding cheat sheet to memorize.
Command bar — inspired by Claude Code’s natural input model
VS Code is 300MB. Vim takes months to learn. cat has no syntax highlighting. There should be a tool that launches in milliseconds, highlights 12 languages, and gets out of your way. That’s edit.
edit — the right tool for the right job

edit brings syntax-highlighted code viewing, git diff review, and instant navigation to every terminal. No configuration. No plugins. Just install and start reading code.

What you get

Everything you need. Nothing you don’t.

Six capabilities in a single binary. Each works out of the box. Together, they’re the fastest way to read and review code in a terminal.

Syntax Highlighting

Tree-sitter powered. 12 languages out of the box — Rust, TypeScript, Python, Go, and more. Incremental parsing that stays fast on large files.

Command Bar

Claude Code-inspired input. Type commands naturally: line numbers, search patterns, symbol names. No modes to memorize. Just type and go.

Git Diff Viewer

Side-by-side diff against git HEAD. Added lines in green, deleted in red. Jump between hunks with F8. See exactly what changed.

Fuzzy File Picker

Nucleo-powered fuzzy matching. Type a few letters to find any file in your project. Respects .gitignore. Opens instantly.

File Tree Sidebar

Collapsible directory tree with git status markers. See which files are modified, added, or untracked at a glance. Toggle with Ctrl-B.

Symbol Navigation

Type goto fn_name and jump to function definitions. Language-aware — knows fn in Rust, def in Python, func in Go.

FAQ

Fair questions, honest answers.

We built edit for ourselves first. These are the questions we asked too.

No. edit is for viewing and navigating code — not writing it from scratch. Think of it as a better cat / less, not a replacement for your main editor. Open a file, read it with syntax highlighting, review a diff, jump to a function, and move on.
edit ships with tree-sitter grammars for 12 languages: Rust, JavaScript, TypeScript, Python, Go, JSON, TOML, YAML, Markdown, Bash, CSS, and HTML. More can be added — PRs welcome.
Nope. Arrow keys navigate. Everything else goes through the command bar — type what you want to do in plain English: 42 to go to line 42, /pattern to search, diff to toggle diff view. Type help if you ever get stuck.
git diff gives you a flat patch in your pager. edit gives you a side-by-side view with syntax highlighting, line numbers, colored additions/deletions, and the ability to jump between hunks with F8. It compares your working copy against git HEAD in real time.
Yes. edit is a single static binary with zero runtime dependencies. Copy it to any server and run it. It works in any terminal that supports 256 colors (true color preferred). No X11, no Wayland, no GUI required.
It’s on the roadmap. The buffer layer already supports insert, delete, and undo (Ropey-based). The current focus is on making the navigation and review experience excellent first. Editing mode will land as an opt-in feature — not a mode switch.

Architecture

edit is built as a Cargo workspace with 8 independent crates:

core-buffer   Ropey-based text buffer with cursor management
core-syntax   Tree-sitter incremental parsing & highlighting
core-diff     Line-based diff engine with git integration
core-theme    VS Code Dark+ inspired color system
core-picker   Nucleo-powered fuzzy file matcher
core-fs      File tree with .gitignore support
ui-tui       Ratatui TUI rendering layer
ui-gui       egui GUI with embedded terminals
app          TUI binary, event loop, keybindings
